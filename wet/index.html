WET

From the WET (Web Experience Toolkit) website:

  • An award-winning front-end framework for building websites that are accessible, usable, interoperable, mobile friendly and multilingual
  • A collection of flexible and themeable templates and reusable components
  • A collaborative open source project led by the Government of Canada

Why Use WET?

  • Accessibility. All the heavy-lifting in that area has been taken care of for you.
  • Consistency accross GoC sites (Internet and Intranet)
  • A whole bunch of pre-made widgets for you to use without having to re-invent the wheel
  • It's open source.

How to Install WET

There are two ways to install WET:

  1. Manually (for anyone)
  2. Using the CDTS (for GoC)

Install WET Manually (Non-GoC)

  1. Go to WET Download Page
  2. Click the "Download" button for "Web Experience Toolkit (WET)".
  3. In the Assets section: click on the wet-boew-dist-#.#.##.zip. This will download the zip file.
  4. Copy and unzip the file somewhere handy on your computer.
  5. This will create a new folder with a lot of files you may not need, including the WET website itself. Here's what you may need. Copy these files/directories to where you need them (ie: your web site's base folder):
    • /theme-wet-boew/ This is where the CSS files and some icons live.
    • /wet-boew/ This is where the Javascript, some other CSS (for polyfills), etc. live
  6. Copy appropriate template files from /theme/. If you want an English page with a section menu (ie: a left-nav menu) copy content-secmenu-en.html to your project folder and update the content to your own.
  7. If your site isn't displaying properly, check your browser's console to see if any WET components are being called, but not found. Copy any missing files from the unzipped WET directory to where they ought to be.

Install WET Manually (GCWeb)

  1. Go to WET Download Page
  2. Click the "Download" button "Canada.ca (GCWeb).
  3. In the Assets section: click on the wet-boew-dist-#.#.##-gcweb.zip. This will download the zip file.
  4. Copy and unzip the file somewhere handy on your computer.
  5. This will create a new folder with a lot of files you may not need, including the WET website itself. Here's what you may need. Copy these files/directories to where you need them:
    • /GCWeb/ This is where the CSS files and some icons live.
    • /wet-boew/ This is where the Javascript, some other CSS (for polyfills), etc. live
  6. Copy /gcweb-theme/static-header-footer/bootstrap-3.html to your website directory as a template file.
  7. Update references to WET CSS and Javascript files to where you copied them.
  8. Copy and paste this template file for each page you need.
  9. Update the content to your own.
  10. If your site isn't displaying properly, check your browser's console to see if any WET components are being called, but not found. Copy any missing files from the unzipped WET directory to where they ought to be.

Find more detailed instructions on the Quick implementation guide - GCWeb theme page.

CDTS - Centrally Deployed Template Solutions

Only for Government of Canada websites.

From the CDTS Website:

The purpose of the CDTS is essentially to deliver the presentation layer of the Canada.ca theme or Intranet.canada.ca theme to web assets.

The CDTS hosts WET and theme files elsewhere. Site owners start with a page template with 2 or 3 <script> elements, some site/page-specific content in some pre-defined JSON variables; and the rest is taken care of for you. You just have to deal with your content.

You can lock into a specific version, or the latest, ie: "run".

How to implement CDTS on your site

There are too many factors and intricacies to list in this tutorial. But in short:

  1. You will need back-up static WET files in case the CDTS host is unreachable, or the user has Javascript turned off, or if it fails somehow. (See instructions earlier in this tutorial for details on how.)
  2. For the rest, start with the CDTS Implementation Guide.
