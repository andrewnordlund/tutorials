WET

From the WET (Web Experience Toolkit) website:

  • An award-winning front-end framework for building websites that are accessible, usable, interoperable, mobile friendly and multilingual
  • A collection of flexible and themeable templates and reusable components
  • A collaborative open source project led by the Government of Canada

Why Use WET?

  • Accessibility. All the heavy-lifting in that area has been taken care of for you.
  • Consistency accross GoC sites (Internet and Intranet)
  • A whole bunch of pre-made widgets for you to use without having to re-invent the wheel
  • It's open source.

How to Install WET

There are two ways to install WET:

  1. Manually (for anyone)
  2. Using the CDTS (for GoC)

Install WET Manually (Non-GoC)

  1. Go to WET Download Page
  2. Click the "Download" button for "Web Experience Toolkit (WET)".
  3. In the Assets section: click on the wet-boew-dist-#.#.##.zip. This will download the zip file.
  4. Copy and unzip the file somewhere handy on your computer.
  5. This will create a new folder with a lot of files you may not need, including the WET website itself. Here's what you may need. Copy these files/directories to where you need them (ie: your web site's base folder):
    • /theme-wet-boew/ This is where the CSS files and some icons live.
    • /wet-boew/ This is where the Javascript, some other CSS (for polyfills), etc. live
  6. Copy appropriate template files from /theme/. If you want an English page with a section menu (ie: a left-nav menu) copy content-secmenu-en.html to your project folder and update the content to your own.
  7. If your site isn't displaying properly, check your browser's console to see if any WET components are being called, but not found. Copy any missing files from the unzipped WET directory to where they ought to be.

See Example Page.

Install WET Manually (GCWeb)

  1. Go to WET Download Page and click the "Download" button "Canada.ca (GCWeb).
  2. In the Assets section: click on the wet-boew-dist-#.#.##-gcweb.zip. This will download the zip file.
  3. Copy and unzip the file somewhere handy on your computer.
  4. This will create a new folder with a lot of files you may not need, including the WET website itself. Here's what you may need. Copy these files/directories to where you need them:
    • /GCWeb/ This is where the CSS files and some icons live.
    • /wet-boew/ This is where the Javascript, some other CSS (for polyfills), etc. live
  5. Copy /gcweb-theme/static-header-footer/bootstrap-3.html to your website directory as a template file.
  6. Update references to WET CSS and Javascript files to where you copied them.
  7. Copy and paste this template file for each page you need.
  8. Update the content to your own.
  9. If your site isn't displaying properly, check your browser's console to see if any WET components are being called, but not found. Copy any missing files from the unzipped WET directory to where they ought to be.

See Example Page.

Find more detailed instructions on the Quick implementation guide - GCWeb theme page.

CDTS - Centrally Deployed Template Solutions

Only for Government of Canada websites.

From the CDTS Website:

The purpose of the CDTS is essentially to deliver the presentation layer of the Canada.ca theme or Intranet.canada.ca theme to web assets.

The CDTS hosts WET and theme files elsewhere. Site owners start with a page template with 2 or 3 <script> elements, some site/page-specific content in some pre-defined JSON variables; and the rest is taken care of for you. You just have to deal with your content.

You can use a specific version, or the latest, ie: "run".

See Example Page.

How to implement CDTS on your site

There are too many factors and intricacies to list in this tutorial. But in short:

  1. You will need back-up static WET files in case the CDTS host is unreachable, or the user has Javascript turned off, or if it fails somehow. (See instructions earlier in this tutorial for details on how.)
  2. Put references to WET CSS and Javascript files inside <noscript> tags.
  3. Call the CDTS with these lines in the <head>: 
    <!-- Load closure template scripts -->
<script src="https://www.canada.ca/etc/designs/canada/cdts/{theme}/v{version_with_underscores}/cdts/compiled/soyutils.js"></script>
<script src="https://www.canada.ca/etc/designs/canada/cdts/{theme}/v{version_with_underscores}/cdts/compiled/wet-en.js"></script>

    Example:
    <!-- Load closure template scripts -->
<script src="https://www.canada.ca/etc/designs/canada/cdts/gcweb/v4_0_28/cdts/compiled/soyutils.js"></script>
<script src="https://www.canada.ca/etc/designs/canada/cdts/gcweb/v4_0_28/cdts/compiled/wet-en.js"></script>
  4. Define the following sections in your template: Head, top, pre-Footer, footer, footer file, and optionally a section menu.

The Outline of a Page

Specifics in each section are defined in JSON objects.

The <head> Section

  • Provide references to static fallback files
  • Set the Adobe Analtics settings
<noscript>
	<!-- Write closure fall-back static file -->
	<!-- /ROOT/etc/designs/canada/cdts/gcweb/v4_0_28/cdts/static/refTop.html -->
	<!--#include virtual="/app/cls/WET/gcweb/v4_0_28/cdts/static/refTop.html" -->
</noscript>
<!-- Write closure template -->
<script>
	document.write(wet.builder.refTop({
		"webAnalytics" : [{
			"environment" : "staging",
			"version" : 1
		}]
	}));
</script>

The Top Section

  • Add a <div id="def-top">. This is everything above the <main>
  • Add a wet.builder.top for breadcrumbs, the language link, etc.

Example:

<div id="def-top">
	<!-- Write closure fall-back static file -->
	<!-- /ROOT/etc/designs/canada/cdts/gcweb/v4_0_28/cdts/static/top-en.html -->
	<!--#include virtual="/app/cls/WET/gcweb/v4_0_28/cdts/static/top-en.html" -->
</div>
<!-- Write closure template -->
<script>
	var defTop = document.getElementById("def-top");
	defTop.outerHTML = wet.builder.top({
		"lngLinks": [{
			"lang": "fr",
			"href": "content-fr.html",
			"text": "Fran√ßais"
		}]
	});
</script>

The Pre-Footer Section

  • Add a <div id="def-prefooter">. This is everything below the main content and the </main> tag.
  • Add a wet.builder.prefooter for feedback links, data modified, screen identifier, etc.

Example:

<div id="def-preFooter">
	<!-- Write closure fall-back static file -->
	<!-- /ROOT/etc/designs/canada/cdts/gcweb/v4_0_28/cdts/static/preFooter-en.html -->
	<!--#include virtual="/app/cls/WET/gcweb/v4_0_28/cdts/static/preFooter-en.html" -->
</div>
<!-- Write closure template -->
<script>
	var defPreFooter = document.getElementById("def-preFooter");
	defPreFooter.outerHTML = wet.builder.preFooter({
		"dateModified": "2017-06-05"
	});
</script>
</main>

The Footer Section

  • Add a <div id="def-footer">. This is everything below the </main> content and the </main> tag.
  • Add a wet.builder.footer for a contact link, features (Intranet), subtheme (Intranet), etc.

  • This is just a Javascript section defining certain variables mostly dealing with leaving a secured site, and some environmental information.

Optional Left-Nav Menu

  • A.K.A.: A Secondary Menu
  • Add "topSecMenu": true in the wet.builder.top section
  • Add an unclosed <div class="container"> and <div class="row"> just before the <main>
  • Immediately after the </main> add a <nav id="wb-sec"> element with fall-back static files in it
  • Add the Javascript to define the contents of the secondary menu
  • Add two </div> to close the two that were added above the <main>

Optional Left-Nav Menu (continued)

Example:

<nav class="wb-sec col-md-3 col-md-pull-9" typeof="SiteNavigationElement" id="wb-sec" role="navigation">
	<!-- Write closure fall-back static file -->
	<!-- /ROOT/etc/designs/canada/cdts/gcweb/v4_0_28/cdts/static/secMenu-en.html -->
	<!--#include virtual="/app/cls/WET/gcweb/v4_0_28/cdts/static/secMenu-en.html" -->
</nav>
<!-- Write closure template -->
<script>
	var secondarymenu = document.getElementById("wb-sec");
	secondarymenu.innerHTML = wet.builder.secmenu({
		"sections": [{
			"sectionName": "[Topic - Local navigation]",
			"menuLinks": [{
				"href": "#",
				"text": "Link 1",
				"subLinks": [{
					"subhref": "#11a",
					"subtext": "Link 1.1 a)"
				}, {
					"subhref": "#11b",
					"subtext": "Link 1.1 b)"
				}]
			}, {
				"href": "#",
				"text": "Link 2"
			}]
		});
</script>

How To Use WET Components

There are two parts to this:

  1. The WET Style Guide - mostly for how to get static content to look a certain way
  2. The WET Working Examples - mostly for how to get dynamic content to work a certain way

Bookmark those pages and refer to them often to see what WET offers, and how to use it.

Most WET components are implemented by adding classes to HTML elements. A lot of the classes come from Bootstrap.

Some WET Styling Examples

  • Remember that <h#> headings are to describe and head content; not to make big, bold test. Use CSS for that.
  • To make something look like an <h2> without altering its semantics, use class="h2" (ex: <div class="h2"> will be big and bold, but won't come up in a list of headings for screen reader users.
  • To expose text to screen readers, but hide it from sight, use class="wb-inv".

See the WET Style Guide for more information.

Some WET Widget Examples

A lot of WET widgets require more than one class and/or more than one element with more than one class; and maybe even some data-* attributes. So, full examples won't be shown in this tutorial, but can be seen in the WET Working Examples.

  • A very important one: Form Validation.
    • Can be hard to get right for accessibility purposes; WET does a lot of the work for you.
    • Always do back-end form validation for security purposes
  • For lists and tables: Filters
  • Session Timeout Widget
  • Footnotes

How to Check For WET/CDTS

The tools to use:

  • For WET, use the WET Widget Version Finder (WWVF) Firefox Extension
    • GitHub Repo
  • For CDTS, use the CDTS Version Finder Firefox Extension (only on GoC network)
    • GCCode Repo
  • These will tell you which version of WET/CDTS a site uses (if any).

How to check for the CDTS and version

  • Use the CDTS Version Funder Firefox Extension.
  • It will tell you if the page you're looking at is using the CDTS, and if so, which version and which theme (GCWeb / GCIntranet)
  • You can see what the page may look like with another version or theme:
    1. Select the desired theme
    2. Select the version
    3. Click "Apply"
    4. Refresh the page in your browser
  • A single page should only be using 1 version and 1 theme.
  • CDTS Issues can be filed on the CDTS issues GitHub repo

How to check for the CDTS and version

How to Check WET Version and Components

  • Use the WWVF (WET Widget Version Finder) Firefox extension.
    • (Note: Due to changes in how versions are dealt with, the WWVF may indicate WET 3.x is being used in some places, when WET 4 is being used.)
  • When looking at a webpage, use View Source (not the Inspector) to see if CDTS sections are implemented properly, and WET classes are used properly.
  • WET classes start with wb- (for WET-BOEW)
  • WET is built on Bootstrap, so a lot of classes, such as .text-center, .text-left, and .col-md-*.
  • You need to compare usage to the WET Working Examples and the WET Style Guide to determine if it's being implemented correctly.
  • If you notice a bug in WET, check the Filing a bug or an issue page.

How to Check WET Version and Components

What to check for with WET

Primarily: If you see a custom widget that could be better accomplished with a WET widget; suggest the WET widget. Note that some sites have requirements that WET doesn't handle; example: an advanced date-picker widget.

If you find an accessibility/WCAG issue on a WET widget:

  1. Ensure that it actually is an issue
  2. Check the version of WET that's being used. Older versions are more likely to have accessibility issues.
    • If a site is using a beta version of an initial release (ex: 3.1.0-b1), then plead with the site owners to upgrade to a more recent version.
    Thre is no (known) requirement to use the latest version of WET, but if upgrading can fix issues, then it should be done if/when possible.
  3. Check with the WET Working Examples and the WET Style Guide to ensure it's being done "the WET way".
  4. Check if the examples on either the WET Working Examples or the WET Style Guide have the same issue.
  5. If the issue exists in the most current version, consider filing a Filing a bug or an issue.

What to check for with CDTS

Note that the CDTS is only available for Government of Canada websites and webapps. A COTS product won't be using it.

  1. Check if a site is using CDTS, and if so, what version.
  2. Only the current version, and two previous versions are officially supported.
  3. Turn off Javascript and reload the page to see if the static fallback files are there.
  4. View Source -> and check for wet.builder.refTop, wet.builder.top, wet.builder.preFooter, wet.builder.footer, wet.builder.refFooter; and if the site uses a left-nav menu: wet.builder.secMenu.
  5. If it's a splash page, check for: wet.builder.splash and wet.builder.splashTop.
